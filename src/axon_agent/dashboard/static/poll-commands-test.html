How it works:
1. The endpoint polls Telegram's getUpdates API for new messages
2. Filters for commands (messages starting with /)
3. Returns command details with metadata (chat_id, timestamp, etc.)
4. Maintains an offset to track processed updates

Test Instructions

  1. Ensure TELEGRAM_BOT_TOKEN is set in your .env file
  2. Start the analytics server: python -m uvicorn analytics_server.server:app --port 8003
  3. Send a command to your Telegram bot (e.g., /status, /pause)
  4. Click "Poll Commands" to fetch the command
  5. The command should appear with full metadata

Supported commands: /status, /pause, /resume, /skip, /priority, /next, /log, /budget
